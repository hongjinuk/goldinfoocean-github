## 1. 빠른 시작 - 로컬에서 테스트

웹사이트를 배포하기 전에 먼저 로컬에서 테스트해보세요.

### 방법 1: Python 사용 (가장 간단)

```bash
# static-site 폴더로 이동
cd static-site

# 웹 서버 시작
python3 -m http.server 8000
```

브라우저에서 접속: http://localhost:8000

### 방법 2: Node.js 사용

```bash
# static-site 폴더로 이동
cd static-site

# serve 패키지 사용 (설치 필요 없음)
npx serve .
```

브라우저에서 표시되는 주소로 접속 (보통 http://localhost:3000)

### 방법 3: PHP 사용

```bash
# static-site 폴더로 이동
cd static-site

# PHP 내장 웹 서버 시작
php -S localhost:8000
```

브라우저에서 접속: http://localhost:8000

## 2. 가장 쉬운 방법 - GitHub Pages (추천) ⭐

GitHub Pages는 **완전 무료**이며 **별도의 링크를 만들 필요 없이** 자동으로 웹사이트 주소를 제공합니다.

### 📍 자동 배포 (권장)

이 저장소는 이미 GitHub Actions가 설정되어 있어, 코드를 푸시하면 **자동으로 배포**됩니다!

#### 배포 확인하기:

1. GitHub 저장소 페이지로 이동
2. 상단의 **Actions** 탭 클릭
3. 가장 최근 워크플로우 실행 확인
4. 완료되면 다음 주소로 접속:
   
   ```
   https://ai-work-project.github.io/goldinfoocean/
   ```

#### 배포 상태 확인:

- ✅ 녹색 체크 표시: 배포 성공
- ❌ 빨간 X 표시: 배포 실패 (워크플로우 로그 확인)
- 🟡 노란 점: 배포 진행 중

### 📍 수동 설정 (Actions가 없는 경우)

1. **GitHub 저장소 설정**
   - 저장소 페이지의 **Settings** 클릭
   - 왼쪽 메뉴에서 **Pages** 선택

2. **Source 설정**
   - Source: **Deploy from a branch** 선택
   - Branch: **gh-pages** 선택 (또는 main)
   - Folder: **/ (root)** 선택
   - **Save** 버튼 클릭

3. **배포 대기**
   - 2-3분 후 페이지 상단에 URL이 표시됩니다
   - 예: `https://ai-work-project.github.io/goldinfoocean/`

4. **사이트 접속**
   - 표시된 URL을 클릭하면 웹사이트가 열립니다!

### 💡 URL 설명

- **기본 형식**: `https://사용자명.github.io/저장소명/`
- **이 프로젝트**: `https://ai-work-project.github.io/goldinfoocean/`
- **별도의 링크를 만들 필요 없이** 이 주소를 공유하면 됩니다!

### 🔄 업데이트 방법

코드를 수정한 후:

```bash
git add .
git commit -m "웹사이트 내용 업데이트"
git push
```

2-3분 후 자동으로 웹사이트에 반영됩니다!

## 3. 다른 무료 배포 옵션

### 🎯 Netlify (초간단 드래그 앤 드롭)

**장점**: 가장 쉬움, 드래그만 하면 끝

1. https://app.netlify.com 접속 및 회원가입
2. "Sites" > "Add new site" > "Deploy manually" 클릭
3. `static-site` 폴더를 드래그 앤 드롭
4. 완료! URL이 자동으로 생성됨 (예: `https://random-name.netlify.app`)

**커스텀 도메인 설정**: Site settings > Domain management

### 🚀 Vercel (개발자 친화적)

**장점**: 빠른 배포, Git 연동

#### 방법 1: CLI 사용

```bash
# Vercel CLI 설치
npm install -g vercel

# static-site 폴더로 이동
cd static-site

# 배포
vercel
```

#### 방법 2: 웹 UI 사용

1. https://vercel.com 접속 및 회원가입
2. "New Project" 클릭
3. GitHub 저장소 연결
4. Root Directory: `static-site` 선택
5. Deploy 클릭

### ☁️ Cloudflare Pages (빠른 속도)

**장점**: 글로벌 CDN, 빠른 속도

1. https://pages.cloudflare.com 접속
2. "Create a project" 클릭
3. GitHub 연결
4. Build settings:
   - Build command: (비워두기)
   - Build output directory: `static-site`
5. Deploy

### 🌊 Surge.sh (명령어 한 줄)

**장점**: 가장 빠름, 명령어 하나면 끝

```bash
# Surge 설치
npm install -g surge

# static-site 폴더로 이동
cd static-site

# 배포
surge
```

첫 실행 시 이메일과 비밀번호 입력 → 완료!

## 4. 배포 후 확인사항

### ✅ 체크리스트

- [ ] 웹사이트가 정상적으로 열리는가?
- [ ] 모든 탭이 잘 작동하는가? (혈액형, MBTI, 호르몬 등)
- [ ] 모바일에서도 잘 보이는가?
- [ ] URL을 복사하여 다른 브라우저에서도 열리는가?

### 📱 모바일 테스트

1. 스마트폰 브라우저에서 URL 입력
2. 또는 QR 코드 생성기로 QR 코드 만들어서 스캔

### 🔗 URL 공유하기

배포가 완료되면 다음과 같이 URL을 공유할 수 있습니다:

- 카카오톡, 메신저 등으로 직접 전송
- SNS에 게시
- QR 코드로 변환하여 오프라인 공유
- 명함에 인쇄

## 5. 문제 해결

### ❓ 404 에러가 나요

**원인**: 파일 경로가 잘못되었거나 배포가 제대로 안 됨

**해결방법**:
1. GitHub Pages: Settings > Pages에서 설정 다시 확인
2. 브랜치가 올바른지 확인
3. 2-3분 기다린 후 다시 시도
4. 브라우저 캐시 삭제 (Ctrl + Shift + R)

### ❓ 스타일이 안 보여요

**원인**: CSS 파일 경로 문제

**해결방법**:
1. `index.html`에서 CSS 경로가 `css/style.css`인지 확인
2. 파일 이름의 대소문자 확인 (Linux는 대소문자 구분)
3. 브라우저 개발자 도구(F12)에서 Network 탭 확인

### ❓ 배포 후 변경사항이 반영 안 돼요

**원인**: 브라우저 캐시 또는 CDN 캐시

**해결방법**:
1. 브라우저 새로고침 (Ctrl + Shift + R 또는 Cmd + Shift + R)
2. 시크릿/프라이빗 모드로 접속
3. 5-10분 기다리기 (CDN 캐시 갱신 시간)
4. 배포 플랫폼에서 강제 재배포

### ❓ "Build failed" 에러가 나요

**원인**: GitHub Actions 설정 문제

**해결방법**:
1. Actions 탭에서 에러 로그 확인
2. `.github/workflows/deploy.yml` 파일 확인
3. 저장소 Settings > Pages > Source를 `gh-pages` 브랜치로 변경

### ❓ 커스텀 도메인을 사용하고 싶어요

**방법**:

1. 도메인 구입 (가비아, Cloudflare 등)
2. DNS 설정:
   - GitHub Pages: CNAME 레코드 추가
   - Netlify/Vercel: 플랫폼 안내에 따라 DNS 설정
3. 플랫폼에서 커스텀 도메인 추가
4. SSL 인증서 자동 발급 대기 (보통 10분 내외)
